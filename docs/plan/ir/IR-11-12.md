# IR-11/12 — Self-healing Contract KB + anti-poison

## Purpose

Contracts converge to observed reality: a Knowledge Base (KB) ingests deviations between declared contracts and real responses, heals a contract only after 3 consistent observations with an identical structural fingerprint, versions with rollback, and never learns from a mock. An API changing shape becomes a detected, healed event instead of a silent permanent breakage. The heal path is fully programmatic, no LLM anywhere in it. This is positioned as the moat: an accumulated record of what APIs do (and where docs lie) does not commoditise as models improve.

## BubbleLab today (the problem)

Nothing. No learning from real responses, no contract memory, no drift signal. When an API changes shape, every flow that touches it breaks forever, silently. The one ingredient BubbleLab does have is identity: `BubbleParser` (`packages/bubble-runtime/src/extraction/BubbleParser.ts`) parses a real AST (`@typescript-eslint/parser`) and assigns distinct per-call-site identity (`hash("421:method#1")`, one bubble used twice gets separate identities). Relevant retraction (HANDOFF §2): the reference build's AST-vs-regex benchmark (F1 0.95 vs 0.34) compared against a naive regex baseline BubbleLab does not use; it is not a delta over BubbleLab. Do not port an identity layer, key the KB on the identity they already assign.

## The delta

New Drizzle tables (`recorded_contracts` keyed by call-site identity with structural fingerprint, response sample, version, source run id; `contract_observations` for the 3-observation heal rule). Ingest points: the Phase-2 tester (test runs) and `bubble-flow-execution.ts` (production runs, consuming an `OUTPUT_MISMATCH` drift code). Recorded samples serve two consumers: `getRecordedMock()` (the merged test-mode switch already ships this seam as `recordedMockProvider`, a designed plug point with fallback) and `get-bubble-details-tool`, so codegen grounds on reality instead of `MockDataGenerator` fiction.

## Why the improvement is needed

Maintenance ceiling and the docs-lie problem. Vendors ship backward-incompatible response changes without notice; a system with no contract memory pays for each one with a production outage per flow, diagnosed by hand. Anti-poison is the other half: a KB that heals on one observation can be poisoned by a transient anomaly or a concurrent mutation into corrupting every flow that shares the contract, so the 3-consistent-observations rule and the never-learn-from-mocks rule are load-bearing, not nice-to-haves.

## Reference build vs BubbleLab port

In `integration_stitcher` the KB was native to the contracts package and its convergence and anti-poison logic passed in the lab, but it never worked in production, see the caveat below.

In BubbleLab the port has two advantages the reference lacked: the call-site identity already exists (nothing to build), and the `recordedMockProvider` seam is already merged. What fights: the drift signal must survive BubbleLab's error handling end-to-end, and the telemetry hard requirement (`TELEMETRY-KIV-AND-SIDE-EFFECT-DETECTION.md` §1) exists precisely because blanket catches flatten typed errors; `BubbleRunner`'s catch-and-push-string error paths are exactly the boundary where a typed code dies today.

## Evidence / status

LAB-ONLY, with a production-blocking caveat that must not be repeated. Convergence and anti-poison both passed in the reference build (it refused to learn from a mocked observation). But in production the KB was starved by a bug: the drift code `OUTPUT_MISMATCH` was thrown by the adapter and collapsed into a generic `PERFORM_FAILED` by a blanket catch at the wrapper boundary, and nothing consumed it, so the KB could only ever learn from test runs. The port therefore carries hard requirements: every failure carries a distinct machine-branchable code, no blanket catch may flatten a typed error, every code has a consumer, and every failure carries call-site identity. Port status: building (Wave 3, branch `improve/contract-kb-and-drift-fix`), sequenced after IR-9/10 because the KB has nothing to eat until reads are grounded.

## Verdict

SURGERY — and only after IR-9/10; preserve the drift code across every boundary and give it a consumer, or this IR starves again.
